Criando um banco de palavras

Realiza a coleta de verbetes desejáveis da Wikipédia. Por desejáveis, entenda-se verbetes com palavras únicas, sem caracteres especiais ou acentos. Um banco com 2000 verbetes foi gerado.

Alimentação de um DB NoSQL (MongoDB) com os verbetes para consulta eficiente dos valores.

Encontrando um modelo de ML

Utilizando o "acerto" ou o "erro" como variável alvo com respeito a um padrão de '__ __ K __ ...' exibido, pôde-se treinar uma máquina para gerar um modelo preditivo eficiente. O modelo de ML escolhico foi o de Árvores de Decisão, que se mostrou eficaz e rápido para o caso. Seguem os parâmetros estatísticos obtidos:

Definindo a função de predição

Realiza a predição da letra com base em um padrão existente, e.g, para a palavra "HOJE": '__ __ J __', e num conjunto de tentativas anteriores [A, P, J]

Contruindo o app

A construção do app, em si, é simples, bastando realizar a pesquisa do verbete na Wiki e extrair os dados úteis, como título e descrição, e então aplicar a função de predição para o Bot.

O app de adivinhação em CLI

Transportando o jogo para aplicação WEB

O framwork Flask foi utilizado para servir a aplicação web e tratar as páginas HTML, apenas ajustando o momento adequado para o Bot e o jogador realizarem a jogada.

Para que não haja conflito nos dados entre jogadores simultâneos, é necessário que se estabeleça uma sessão para cada um quando existe o acesso.
